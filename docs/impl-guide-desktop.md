# Implementation Guide — Desktop Support

Covers Phase 4b: adding macOS, Windows, Linux (and Android) platform support, responsive layout, keyboard shortcuts, and desktop interaction polish.

---

## IG20: Platform Scaffolding

### IG20.1 Generate Platform Directories

```bash
flutter create --platforms=macos,windows,linux,android .
```

This generates `macos/`, `windows/`, `linux/`, and `android/` directories with default runner templates. The existing `ios/` directory is unaffected.

### IG20.2 macOS BLE Entitlements

The macOS app needs Bluetooth entitlements to use `universal_ble`. Edit both entitlements files:

**`macos/Runner/DebugProfile.entitlements`** — add inside the `<dict>`:
```xml
<key>com.apple.security.device.bluetooth</key>
<true/>
```

**`macos/Runner/Release.entitlements`** — same key.

**`macos/Runner/Info.plist`** — add inside the `<dict>`:
```xml
<key>NSBluetoothAlwaysUsageDescription</key>
<string>Wattalizer connects to your power meter and heart rate sensor to record training data.</string>
```

macOS sandboxing remains enabled. The bluetooth entitlement works within the sandbox.

### IG20.3 Windows BLE

`universal_ble` uses WinRT on Windows. No special manifest changes needed. **Minimum requirement: Windows 10 version 1803+.** The default `windows/runner/main.cpp` generated by Flutter works as-is.

### IG20.4 Linux BLE

`universal_ble` uses BlueZ on Linux via D-Bus. The default `linux/my_application.cc` works as-is. **Runtime dependency: BlueZ must be installed** (`bluez` package on most distros).

### IG20.5 Android BLE Permissions

**`android/app/src/main/AndroidManifest.xml`** — add inside `<manifest>`:
```xml
<uses-permission android:name="android.permission.BLUETOOTH_SCAN" />
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT" />
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION" />
<uses-permission android:name="android.permission.BLUETOOTH_ADVERTISE" />
<uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
```

### IG20.6 Minimum Window Size (Desktop)

Set minimum window size to 400×600 in each platform's runner:

**macOS** — `macos/Runner/MainFlutterWindow.swift`:
```swift
self.setContentMinSize(NSSize(width: 400, height: 600))
```

**Windows** — `windows/runner/main.cpp`: modify `Win32Window::Create` or add `SetWindowPos` with minimum size.

**Linux** — `linux/my_application.cc`:
```c
gtk_widget_set_size_request(GTK_WIDGET(window), 400, 600);
```

### IG20.7 Verification

```bash
flutter run -d macos     # verify app launches, BLE scan works
flutter run -d windows   # verify on Windows if available
flutter run -d linux     # verify on Linux if available
flutter test             # all existing tests still pass
```

## IG21: Package Compatibility

### IG21.1 wakelock_plus

`wakelock_plus` supports iOS, Android, macOS, and Windows. **Linux is not supported.** Add a platform guard in `lib/domain/services/ride_session_manager.dart`:

```dart
import 'dart:io' show Platform;

// In the constructor, modify the default wakelock callbacks:
final _enableWakelock = enableWakelock ??
    (Platform.isLinux ? () async {} : WakelockPlus.enable);
final _disableWakelock = disableWakelock ??
    (Platform.isLinux ? () async {} : WakelockPlus.disable);
```

This is a 2-line change. The injectable architecture already supports this cleanly.

### IG21.2 share_plus

Not yet used in the codebase. When implementing the export UI (Phase 4, item 19), use a platform-aware approach:
- **Mobile (iOS/Android)**: `Share.shareXFiles()` to open the native share sheet.
- **Desktop (macOS/Windows/Linux)**: `FilePicker.platform.saveFile()` to let the user choose a save location, then copy the TCX file there.

## IG22: Responsive Layout

### IG22.1 Breakpoints

Create `lib/presentation/layout/breakpoints.dart`:

```dart
/// Material 3 canonical breakpoints for responsive layout.
enum LayoutSize { compact, medium, expanded }

/// Determine layout size from available width.
LayoutSize layoutSizeOf(double width) {
  if (width < 600) return LayoutSize.compact;
  if (width < 900) return LayoutSize.medium;
  return LayoutSize.expanded;
}
```

| Layout | Width | Navigation | Ride Screen |
|--------|-------|-----------|-------------|
| Compact | < 600dp | Bottom nav (or menu) | Focus OR Chart (toggle) |
| Medium | 600–899dp | Navigation rail | Focus OR Chart (toggle) |
| Expanded | ≥ 900dp | Navigation rail | Focus + Chart side-by-side |

### IG22.2 AdaptiveShell

Create `lib/presentation/layout/adaptive_shell.dart`:

```dart
import 'package:flutter/material.dart';
import 'breakpoints.dart';
import '../screens/ride_screen.dart';

class AdaptiveShell extends StatefulWidget {
  const AdaptiveShell({super.key});

  @override
  State<AdaptiveShell> createState() => _AdaptiveShellState();
}

class _AdaptiveShellState extends State<AdaptiveShell> {
  int _selectedIndex = 0;

  // Screens for each tab. Stubs for unimplemented screens.
  static const _screens = <Widget>[
    RideScreen(),       // 0: Ride
    Placeholder(),      // 1: History (Phase 4, item 17)
    Placeholder(),      // 2: PDC (Phase 4, item 18)
    Placeholder(),      // 3: Settings (Phase 4, item 19)
  ];

  static const _destinations = <NavigationDestination>[
    NavigationDestination(icon: Icon(Icons.directions_bike), label: 'Ride'),
    NavigationDestination(icon: Icon(Icons.history), label: 'History'),
    NavigationDestination(icon: Icon(Icons.show_chart), label: 'PDC'),
    NavigationDestination(icon: Icon(Icons.settings), label: 'Settings'),
  ];

  static const _railDestinations = <NavigationRailDestination>[
    NavigationRailDestination(icon: Icon(Icons.directions_bike), label: Text('Ride')),
    NavigationRailDestination(icon: Icon(Icons.history), label: Text('History')),
    NavigationRailDestination(icon: Icon(Icons.show_chart), label: Text('PDC')),
    NavigationRailDestination(icon: Icon(Icons.settings), label: Text('Settings')),
  ];

  @override
  Widget build(BuildContext context) {
    return LayoutBuilder(
      builder: (context, constraints) {
        final layout = layoutSizeOf(constraints.maxWidth);

        if (layout == LayoutSize.compact) {
          return Scaffold(
            body: _screens[_selectedIndex],
            bottomNavigationBar: NavigationBar(
              selectedIndex: _selectedIndex,
              onDestinationSelected: (i) => setState(() => _selectedIndex = i),
              destinations: _destinations,
            ),
          );
        }

        // Medium or Expanded: use NavigationRail
        return Scaffold(
          body: Row(
            children: [
              NavigationRail(
                selectedIndex: _selectedIndex,
                onDestinationSelected: (i) => setState(() => _selectedIndex = i),
                labelType: NavigationRailLabelType.all,
                destinations: _railDestinations,
              ),
              const VerticalDivider(thickness: 1, width: 1),
              Expanded(child: _screens[_selectedIndex]),
            ],
          ),
        );
      },
    );
  }
}
```

### IG22.4 RideScreen Layout Adaptation

In `lib/presentation/screens/ride_screen.dart`, replace `OrientationBuilder` with `LayoutBuilder` in `_ActiveView`:

```dart
class _ActiveView extends StatelessWidget {
  const _ActiveView({required this.state, required this.ref});

  final RideStateActive state;
  final WidgetRef ref;

  @override
  Widget build(BuildContext context) {
    return GestureDetector(
      onHorizontalDragEnd: (details) {
        if ((details.primaryVelocity ?? 0).abs() > 200) {
          ref.read(rideModeProvider.notifier).toggle();
        }
      },
      child: LayoutBuilder(
        builder: (context, constraints) {
          final layout = layoutSizeOf(constraints.maxWidth);

          if (layout == LayoutSize.expanded) {
            // Side-by-side: Focus + Chart, no mode toggle needed
            return Row(
              children: [
                Expanded(
                  flex: 2,
                  child: _FocusMode(state: state, ref: ref),
                ),
                Expanded(
                  flex: 3,
                  child: _ChartMode(state: state, ref: ref, isLandscape: false),
                ),
              ],
            );
          }

          // Compact or medium: single panel with mode toggle
          // Also handle mobile landscape (use chart)
          final orientation = MediaQuery.orientationOf(context);
          if (orientation == Orientation.landscape) {
            return _ChartMode(state: state, ref: ref, isLandscape: true);
          }
          final mode = ref.watch(rideModeProvider);
          return mode == RideMode.focus
              ? _FocusMode(state: state, ref: ref)
              : _ChartMode(state: state, ref: ref, isLandscape: false);
        },
      ),
    );
  }
}
```

Key changes:
- `OrientationBuilder` → `LayoutBuilder` + `MediaQuery.orientationOf(context)`
- Width ≥ 900: `Row` with Focus (flex: 2) + Chart (flex: 3)
- Width < 900: current behavior (orientation check + mode toggle)
- Mobile landscape (typically 600–900dp) naturally falls into single-panel chart

### IG22.5 Adaptive Device Connection

In `lib/presentation/widgets/device_sheet.dart`, make `showDeviceSheet()` adaptive:

```dart
void showDeviceSheet(BuildContext context) {
  final width = MediaQuery.sizeOf(context).width;

  if (width < 600) {
    // Compact: bottom sheet (current behavior)
    showModalBottomSheet<void>(
      context: context,
      isScrollControlled: true,
      backgroundColor: Theme.of(context).colorScheme.surface,
      shape: const RoundedRectangleBorder(
        borderRadius: BorderRadius.vertical(top: Radius.circular(16)),
      ),
      builder: (_) => DraggableScrollableSheet(
        initialChildSize: 0.6,
        minChildSize: 0.3,
        maxChildSize: 0.9,
        expand: false,
        builder: (context, scrollController) =>
            _DeviceSheetContent(scrollController: scrollController),
      ),
    );
  } else {
    // Medium/Expanded: dialog
    showDialog<void>(
      context: context,
      builder: (_) => Dialog(
        child: ConstrainedBox(
          constraints: const BoxConstraints(maxWidth: 480, maxHeight: 600),
          child: _DeviceSheetContent(scrollController: ScrollController()),
        ),
      ),
    );
  }
}
```

Extract the existing sheet body into a `_DeviceSheetContent` widget that accepts a `ScrollController` parameter. The content is the same for both presentations.

## IG23: Keyboard Shortcuts

### IG23.1 Intent Classes

Create `lib/presentation/layout/keyboard_shortcuts.dart`:

```dart
import 'package:flutter/material.dart';
import 'package:flutter/services.dart';

// Intents for ride screen actions
class SetFocusModeIntent extends Intent {
  const SetFocusModeIntent();
}

class SetChartModeIntent extends Intent {
  const SetChartModeIntent();
}

class ManualLapIntent extends Intent {
  const ManualLapIntent();
}

class StopRideIntent extends Intent {
  const StopRideIntent();
}

class StartRideIntent extends Intent {
  const StartRideIntent();
}

class OpenDeviceSheetIntent extends Intent {
  const OpenDeviceSheetIntent();
}

/// Keyboard shortcut map for the active ride screen.
const activeRideShortcuts = <ShortcutActivator, Intent>{
  SingleActivator(LogicalKeyboardKey.arrowLeft): SetFocusModeIntent(),
  SingleActivator(LogicalKeyboardKey.arrowRight): SetChartModeIntent(),
  SingleActivator(LogicalKeyboardKey.space): ManualLapIntent(),
  SingleActivator(LogicalKeyboardKey.escape): StopRideIntent(),
  SingleActivator(LogicalKeyboardKey.keyD): OpenDeviceSheetIntent(),
};

/// Keyboard shortcut map for the idle ride screen.
const idleRideShortcuts = <ShortcutActivator, Intent>{
  SingleActivator(LogicalKeyboardKey.enter): StartRideIntent(),
  SingleActivator(LogicalKeyboardKey.keyD): OpenDeviceSheetIntent(),
};
```

### IG23.2 Wiring Shortcuts into RideScreen

Wrap `_ActiveView` in `Shortcuts` + `Actions` + `Focus`:

```dart
// In _ActiveView.build(), wrap the GestureDetector:
Shortcuts(
  shortcuts: activeRideShortcuts,
  child: Actions(
    actions: <Type, Action<Intent>>{
      SetFocusModeIntent: CallbackAction<SetFocusModeIntent>(
        onInvoke: (_) {
          ref.read(rideModeProvider.notifier).setFocus();
          return null;
        },
      ),
      SetChartModeIntent: CallbackAction<SetChartModeIntent>(
        onInvoke: (_) {
          ref.read(rideModeProvider.notifier).setChart();
          return null;
        },
      ),
      ManualLapIntent: CallbackAction<ManualLapIntent>(
        onInvoke: (_) {
          ref.read(rideSessionProvider.notifier).manualLap();
          return null;
        },
      ),
      StopRideIntent: CallbackAction<StopRideIntent>(
        onInvoke: (_) {
          _showStopConfirmation(context);
          return null;
        },
      ),
      OpenDeviceSheetIntent: CallbackAction<OpenDeviceSheetIntent>(
        onInvoke: (_) {
          showDeviceSheet(context);
          return null;
        },
      ),
    },
    child: Focus(
      autofocus: true,
      child: /* existing GestureDetector tree */,
    ),
  ),
)
```

### IG23.3 Stop Ride Confirmation Dialog

When stop is triggered by Escape key, show a confirmation dialog instead of the long-press interaction:

```dart
Future<void> _showStopConfirmation(BuildContext context) async {
  final confirmed = await showDialog<bool>(
    context: context,
    builder: (ctx) => AlertDialog(
      title: const Text('Stop Ride?'),
      content: const Text('Are you sure you want to end this ride?'),
      actions: [
        TextButton(
          onPressed: () => Navigator.pop(ctx, false),
          child: const Text('Cancel'),
        ),
        TextButton(
          onPressed: () => Navigator.pop(ctx, true),
          child: const Text('Stop'),
        ),
      ],
    ),
  );
  if (confirmed == true) {
    ref.read(rideSessionProvider.notifier).endRide();
  }
}
```

### IG23.5 Focus Management

Critical: the `Focus` widget with `autofocus: true` must be in the tree for keyboard events to be received. After dismissing a dialog (stop confirmation, device sheet), the focus may be lost. Handle this by requesting focus back:

```dart
// After dialog dismissal:
if (mounted) {
  FocusScope.of(context).requestFocus();
}
```

On mobile without a physical keyboard, the `Shortcuts` widget is inert — no events fire, no side effects. No platform check needed.

## IG24: Desktop Interaction Polish

### IG24.1 Mouse Cursors

Wrap clickable elements with `MouseRegion` for pointer cursor:

```dart
MouseRegion(
  cursor: SystemMouseCursors.click,
  child: GestureDetector(
    onTap: () { /* action */ },
    child: /* button content */,
  ),
)
```

Apply to: LAP button, STOP button, mode toggle buttons, sensor status bar, device list items.

Alternatively, replace raw `GestureDetector` with `InkWell` which provides both the pointer cursor and hover ripple automatically.

### IG24.2 Tooltips

Wrap interactive elements with `Tooltip`:

```dart
Tooltip(
  message: 'Manual lap (Space)',
  child: /* LAP button */,
)
```

Tooltip text for each element:

| Element | Tooltip |
|---------|---------|
| LAP button | "Manual lap (Space)" |
| STOP button | "Stop ride (hold 1.5s or Esc)" |
| Focus mode pill | "Focus mode (←)" |
| Chart mode pill | "Chart mode (→)" |
| Sensor status bar | "Manage devices (D)" |
| Start button | "Start ride (Enter)" |

### IG24.3 Hover States

Replace the custom `_ModeButton` (currently `GestureDetector` + `Container`) with `InkWell` for automatic hover highlighting:

```dart
// Before:
GestureDetector(
  onTap: onTap,
  child: Container(
    decoration: BoxDecoration(/* ... */),
    child: Text(label),
  ),
)

// After:
InkWell(
  onTap: onTap,
  borderRadius: BorderRadius.circular(16),
  child: Container(
    decoration: BoxDecoration(/* ... */),
    child: Text(label),
  ),
)
```

### IG24.4 Scrollbar Behavior

Flutter automatically shows scrollbars on desktop platforms. No explicit configuration needed. If custom styling is desired, configure `ScrollbarThemeData` in the app's `ThemeData`.
